## 1. 시작하기

### 앱 실행

빌드된 EXE를 직접 실행합니다.

```
src\GameDataViewer.App\bin\Debug\net8.0-windows\GameDataViewer.exe
```

또는 배포용 단일 EXE를 사용할 경우:

```
publish\win-x64\GameDataViewer.exe
```

> **참고:** `dotnet run` 명령은 이 환경에서 앱 제어 정책으로 인해 차단될 수 있습니다. 반드시 EXE를 직접 실행해주세요.

### 초기 데이터 로드

앱 최초 실행 시 자동으로 다음 작업이 수행됩니다:

1. SQLite 데이터베이스 생성 (`%LOCALAPPDATA%\GameDataViewer\gamedata.db`)
2. 시드 데이터 적재 — 아이템 30건, 스킬 25건, 몬스터 20건
3. 각 탭 데이터 자동 로드

로딩 중에는 하단 상태바에 원형 스피너(ProgressRing)가 표시됩니다.

## 2. 화면 구성

```
┌──────────────────────────────────────────────────────┐
│  ⚔ 아이템  ✨ 스킬  👾 몬스터  📊 통계  ⚙ 설정      │  ← 탭 헤더 (선택된 탭: 파란 상단 선)
├──────────────────────────────────────────────────────┤
│  [검색창]  [검색]  [등급▼]  [카테고리▼]  [CSV↓][XLS↓] │  ← 필터 바
├────────────────────────────┬─────────────────────────┤
│                            │                         │
│  데이터 그리드 (목록)       │  상세 정보 패널          │
│                            │                         │
├────────────────────────────┴─────────────────────────┤
│  ◀ 이전    페이지 1 / 1    다음 ▶                     │  ← 페이징 바
├──────────────────────────────────────────────────────┤
│  상태 메시지                              [스피너]    │  ← 상태 바
└──────────────────────────────────────────────────────┘
```

| 영역 | 설명 |
|------|------|
| **탭 헤더** | 5개 기능 탭 — 선택된 탭은 파란 상단 테두리 + 흰색 Bold 텍스트 |
| **필터 바** | 검색 · 등급/분류 필터 · CSV/Excel 내보내기 버튼 |
| **데이터 그리드** | 페이징 처리된 데이터 목록. 컬럼 헤더 클릭 시 정렬 |
| **상세 정보 패널** | 행 선택 시 우측에 상세 속성 표시 |
| **페이징 바** | 이전/다음 페이지 이동, 현재 페이지/총 페이지 수 표시 |
| **상태 바** | 로드 완료 건수, 로딩 중 스피너 |

## 3. 아이템 탭 ⚔

게임 내 아이템(무기·방어구·장신구·소모품·재료) 전체 목록을 조회합니다.

### 필터 기능

| 필터 | 설명 | 예시 |
|------|------|------|
| 이름 검색 | 아이템 이름 부분 일치 검색 | `검` → "혼돈의 소검", "수련자의 검" |
| 등급 | 드롭다운 선택 — 전설/영웅/희귀/일반 | `전설` → 전설 등급만 표시 |
| 카테고리 | 무기/방어구/장신구/소모품/재료 | `무기` → 무기류만 표시 |

### 컬럼 설명

| 컬럼 | 설명 |
|------|------|
| ID | 아이템 고유 식별번호 |
| 이름 | 아이템 명칭 |
| 등급 | 일반 / 희귀 / 영웅 / 전설 / 유물 |
| 분류 | 무기 / 방어구 / 장신구 / 소모품 / 재료 |
| 공격력 | 물리 공격력 수치 |
| 방어력 | 물리 방어력 수치 |
| 마력 | 마법 공격/방어 수치 |
| 필요LV | 착용 최소 레벨 |
| 교환 | ☑ 교환 가능 / ☐ 교환 불가 |
| 가격 | 상점 판매 가격 (단위: 골드) |

### 행 색상 의미

| 색상 | 등급 |
|------|------|
| 주황빛 배경 | 전설 |
| 파란빛 배경 | 영웅 |
| 초록빛 배경 | 고급(희귀) |
| 기본 (배경 없음) | 일반 |

### 상세 정보 패널

목록에서 행을 클릭하면 우측 패널에 표시됩니다:
- 등급 색상 뱃지 (등급별 색상 구분)
- 6개 스탯 수치 (공격력 / 방어력 / 마력 / 필요레벨 / 가격 / 교환가능 여부)
- 아이템 설명 텍스트

## 4. 스킬 탭 ✨

직업별 스킬 목록을 조회합니다.

### 필터 기능

| 필터 | 설명 | 예시 |
|------|------|------|
| 이름 검색 | 스킬 이름 부분 일치 | `드래곤` → "드래곤 브레스" |

### 컬럼 설명

| 컬럼 | 설명 |
|------|------|
| 직업 | 스킬을 사용할 수 있는 캐릭터 직업 |
| 유형 | 액티브 (직접 사용) / 패시브 (상시 발동) |
| 속성 | 화 / 수 / 풍 / 지 / 빛 / 암 / 무 |
| 최소데미지 | 스킬 최소 데미지 |

| 궁극기 | ☑ = 궁극기 스킬 (주황빛 행 강조) |

### 상세 정보 패널

- 속성 색상 뱃지 (화=빨강, 수=파랑, 풍=초록 등)

- 데미지 범위, MP 소모, 쿨다운, 사정거리
- 스킬 설명 텍스트

## 6. 통계 탭 📊

전체 데이터를 시각화한 4개의 차트를 제공합니다.

```
┌──────────────────┬──────────────────┐
│  아이템 등급 분포 │  몬스터 등급 분포 │  (파이차트)
├──────────────────┼──────────────────┤
│  스킬 최대데미지  │  몬스터 HP TOP10  │  (수평 바차트)
│  TOP 10          │                  │
└──────────────────┴──────────────────┘
```

| 차트 | 설명 |
|------|------|
| **아이템 등급 분포** | 전설/영웅/희귀/일반 등급별 아이템 비율 (파이차트) |
| **몬스터 등급 분포** | 일반/엘리트/보스/레이드 등급별 몬스터 비율 (파이차트) |
| **스킬 최대데미지 TOP10** | 최대 데미지 기준 상위 10개 스킬 비교 (수평 바차트) |
| **몬스터 HP TOP10** | HP 기준 상위 10개 몬스터 비교 (수평 바차트) |

> 통계 탭은 앱 실행 시 자동으로 데이터를 로드합니다. 차트 위에 마우스를 올리면 수치가 표시됩니다.

## 7. 설정 탭 ⚙

앱 동작 방식을 사용자가 직접 조정할 수 있습니다. 설정은 `%LOCALAPPDATA%\GameDataViewer\settings.json`에 저장됩니다.

### 일반

| 항목 | 설명 | 기본값 | 범위 |
|------|------|--------|------|
| **기본 페이지 크기** | 각 탭의 목록에서 한 번에 표시할 행 수 | 50 | 10 ~ 500 |
| **검색 디바운스** | 검색어 입력 후 실제 검색이 시작되기까지의 대기 시간 (ms) | 300ms | **최솟값 50ms** ~ 2000ms |

> **기본 페이지 크기**: 값을 크게 하면 더 많은 데이터를 한 화면에 볼 수 있지만, 로딩 속도가 느려질 수 있습니다.  
> **검색 디바운스**: 타이핑 도중 불필요한 검색을 줄이는 장치입니다. 50ms = 입력 즉시 반응, 300ms = 0.3초 후 반응, 1000ms = 1초 후 반응. 값이 작을수록 빠르게 반응하지만 타이핑 중 DB 부하가 증가합니다.

### 테마

다크/라이트 계열 15개 테마 중 선택할 수 있습니다. 선택 즉시 미리보기가 적용되며, **💾 저장** 버튼을 눌러야 영구 적용됩니다.

### 내보내기

CSV 또는 Excel 파일을 저장할 기본 폴더를 지정합니다. 비워두면 `내 문서` 폴더가 사용됩니다. 📂 탐색 버튼으로 폴더를 선택할 수 있습니다.

### 디버그

디버그 로그를 활성화합니다. 로그 파일은 `%LOCALAPPDATA%\GameDataViewer\logs\app-YYYYMMDD.log`에 저장됩니다. 다음 실행부터 적용됩니다.

### 버튼

| 버튼 | 동작 |
|------|------|
| **💾 저장** | 현재 입력된 설정을 파일에 저장 |
| **↩ 기본값 복원** | 모든 설정을 초기 기본값으로 되돌림 |

## 8. 데이터 내보내기

아이템, 스킬, 몬스터 탭 필터 바 우측에 내보내기 버튼이 있습니다.

```
[이름 검색...]  [검색]  [등급▼]  [카테고리▼]  |  [CSV ↓]  [XLS ↓]
```

### 동작 방식

1. 현재 적용된 **검색어 · 등급 · 카테고리 필터를 그대로 유지**하면서 전체 데이터를 내보냅니다 (최대 10,000건).
2. 버튼 클릭 시 파일 저장 대화상자가 열립니다.
3. 저장 경로는 설정 탭의 **기본 저장 폴더** 기준으로 초기 위치가 설정됩니다.

### 파일 형식

| 버튼 | 형식 | 용도 |
|------|------|------|
| **CSV ↓** | `.csv` (UTF-8) | 메모장, 엑셀, 모든 스프레드시트에서 열기 가능 |
| **XLS ↓** | `.xlsx` | Microsoft Excel 전용 — 헤더 Bold + 자동 컬럼 너비 적용 |

## 9. 설정 항목 상세 설명

### 기본 페이지 크기란?

데이터 목록(아이템·스킬·몬스터)에서 **한 페이지에 보여줄 행의 수**입니다.

- 예) 기본값 50: 아이템 30건이므로 1페이지에 전부 표시 → 페이징 버튼 비활성화
- 예) 10으로 설정: 30건 ÷ 10 = 3페이지 → 이전/다음 버튼으로 페이지 이동 필요
- 실무에서는 대량 데이터(수천 건) 환경에서 50~100을 권장합니다.

### 검색 디바운스란?

검색창에 글자를 입력할 때마다 즉시 DB를 조회하면 불필요한 부하가 발생합니다.  
디바운스는 **"마지막 입력 후 N밀리초가 지나야 검색을 실행"** 하는 지연 장치입니다.

| 설정값 | 체감 | 상황 |
|--------|------|------|
| 50ms | 거의 즉각 반응 | 소량 데이터, 빠른 응답 필요 시 |
| 300ms (기본) | 0.3초 후 반응 | 일반 사용 권장 |
| 1000ms | 1초 후 반응 | 대용량 DB, 서버 부하 최소화 필요 시 |

> **최솟값은 50ms**입니다. 50ms 미만으로 설정할 수 없습니다.

## 10. 자주 묻는 질문 (FAQ)

**Q. 앱 실행 시 데이터가 보이지 않습니다.**  
A. 첫 실행 시 자동으로 시드 데이터를 적재합니다. 수 초 후 데이터가 표시됩니다. 하단 상태바에 스피너가 사라지면 로드 완료입니다.

**Q. `dotnet run` 명령으로 실행이 안됩니다.**  
A. Windows 앱 제어 정책으로 인해 동적 DLL 로딩이 차단될 수 있습니다. 빌드 후 `GameDataViewer.exe`를 직접 실행해주세요.

**Q. 탭이 보이지 않습니다.**  
A. 창을 클릭하거나 잠시 기다려주세요. `mainWindow.Activate()`가 호출되어 즉시 표시됩니다. 증상이 지속되면 재빌드 후 실행해주세요.

**Q. CSV 내보내기를 했는데 한글이 깨집니다.**  
A. 엑셀에서 열 때 인코딩을 **UTF-8**로 선택해야 합니다. 파일 → 가져오기 → 텍스트 파일 → 구분기호: 쉼표 → 파일 원본: 65001(UTF-8) 선택.

**Q. 로그 파일은 어디에 있나요?**  
A. `%LOCALAPPDATA%\GameDataViewer\logs\app-YYYYMMDD.log` (날짜별 자동 롤링, 7일 보관)

**Q. 데이터베이스 파일 위치는?**  
A. `%LOCALAPPDATA%\GameDataViewer\gamedata.db`  
   앱 최초 실행 시 자동 생성됩니다.

**Q. 실제 게임 서버 API와 연동할 수 있나요?**  
A. `src\GameDataViewer.Infrastructure\ApiClients\LineageMApiClient.cs`의 엔드포인트를 실제 API 주소로 수정하고, `App.xaml.cs`의 주석 처리된 `AddHttpClient` 블록을 해제하면 활성화됩니다.
